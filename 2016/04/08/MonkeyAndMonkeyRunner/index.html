MonkeyAndMonkeyRunner

  • test

一. Monkey与MonkeyRunner的不同点

  1. monkey运行在设备或模拟器上，可以脱离PC运行；monkeyRunner运行在PC上，需要通过服务器/客户端的模式向设备或模拟器上的Android应用发送指令来执行测试。
  2. 普遍的做法是将monnkey作为一个向待测应用发送随机按键消息的测试工具，验证待测应用在这些随机事件的输入面前是否闪退或崩溃。而mmonkeyrunner则接受一个明确的测试脚本（Python编写）。
  3. monkey不支持条件判断，也不支持读取待测界面的信息来执行验证操作,monkeyrunner 的测试脚本中有明确的条件判断语句，可以用来做功能测试。monkeyrunner允许在测试中通过坐标位置、控件ID和控件上的文字操作应用的界面元素。

二. MonkeyRunner用例

  1. monkeyrunner脚本一般格式
    #在程序中引入monkeyrunner模块
    from com.android.monkeyrunner import
    #连接正在运行的设备
    MonkeyRunner,MonkeyDevice,MonkeyImage
    #安装待测应用
    device = MonkeyRunner.waitForConnection()
    device.instalPackage(‘…/.apk’)
    package = ‘’
    activity = ‘’
    #设置要启动的活动类名，由包名和活动类型全名组成
    runComponent = package + ‘/‘ + activity
    #启动活动组件
    device.startActivity(component=runComponent)
    #单击菜单按钮
    device.press(‘KEYCODE_MENU’,MonkeyDevice.DOWN_AND_UP)
    #给设备截图
    result = device.takeSnapshot()
    #保存截图
    result.writeToFile(‘…/.png’,’png’)
